Virtual DOM

React DOM compares the element and its child tree with the previous version and makes only the minimally necessary changes to the DOM. Instead of interacting with the DOM directly, we work with its lightweight copy. We can make changes to the copy based on our needs, and then apply the changes to the real DOM. In this case, the DOM tree is compared with its virtual copy, the difference is determined and the redrawing of what has been changed is started.

The state gives the components the ability to respond to user actions, server responses, and other events without compromising the purity of the component. React components have a built-in state object. This state object stores property values that belong to the component. When the state object changes, the component is redrawn. ------>>>>> State object is initialized in the constructor.
